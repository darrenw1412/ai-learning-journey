Application Programming Interface
API is basically the middleman between 2 software where, one of the software is being the one that requests the data from the other website. For example our smartphone is requesting data from the weather.com to have the data. (Why did Samsung didn't use their own tools to collect the data?) what for? because there's already a service that provide the data and you didn't need to do the hard work where you can just request the data right? Basically, the analogy is, you are requesting on the table reservation on a restaurant and this restaurant will check is your request is approvable or no, in this case you are the one who makes the request and the restaurant is providing the service. Let's think if this kind of service doesn't exist. You are suppose to do all of the work right? you can even access to the restaurant data, kitchen staff, etc. and the restaurant doesn't want that to happen right? So this kind of service is exist and API using this concepts.
